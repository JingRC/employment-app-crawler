# 采集器代码目录结构与模块模板

## 推荐目录结构

```text
collector_service/
  adapters/
    company_example.py
  pipeline/
    deduplicator.py
    normalizer.py
    notifier.py
  spiders/
    base_spider.py
    dynamic_spider.py
    html_spider.py
    json_spider.py
  models.py
  runner.py
```

## 各模块职责

### spiders/base_spider.py

定义采集器基础接口：

1. `fetch()` 获取原始内容
2. `parse()` 解析原始内容
3. `normalize()` 输出统一字段
4. `run()` 执行完整流程

### spiders/html_spider.py

适合静态页面：

1. requests
2. BeautifulSoup
3. lxml/XPath

### spiders/json_spider.py

适合接口型采集：

1. requests/httpx
2. Fiddler 抓包后复现接口
3. JSONPath 提取字段

### spiders/dynamic_spider.py

适合前端渲染：

1. Playwright
2. 获取页面 HTML 或接口响应

### adapters/company_example.py

每个公司/来源一个适配器，只处理站点差异，不修改主流程。

### pipeline/normalizer.py

负责字段标准化：

1. 时间格式统一
2. 城市名统一
3. 链接清洗
4. 空白字符清理

### pipeline/deduplicator.py

负责：

1. 职位唯一标识生成
2. 是否新增判断
3. 是否更新判断

### pipeline/notifier.py

负责：

1. 匹配收藏企业
2. 匹配收藏搜索条件
3. 生成提醒消息

### runner.py

定时任务入口：

1. 选择数据源
2. 执行采集
3. 写入数据库
4. 触发通知

## 使用建议

1. 新增一个数据源时，优先写一个 adapter，而不是直接改主爬虫。
2. 所有来源最终都要输出统一字段结构。
3. 先保证增量采集可用，再考虑复杂分页与风控问题。
4. 先做官网招聘页，再扩展公开招聘站。

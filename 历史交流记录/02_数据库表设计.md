# 就业聚合 App 数据库表设计

## 设计目标

数据库设计需要支持以下能力：

1. 多来源职位统一存储
2. 职位去重与更新检测
3. 收藏企业、职位、搜索条件
4. 提醒推送与已读状态
5. 官网链接和来源链接并存

推荐数据库：PostgreSQL

## 去重与更新策略

### 唯一标识建议

优先级如下：

1. 使用来源自带的职位 ID
2. 若没有，则使用以下字段生成哈希：

```text
source_code + company_name + title + city_name + source_url
```

### 更新检测

每次抓取时对以下内容计算 `content_hash`：

1. title
2. salary_text
3. degree_text
4. experience_text
5. description_text
6. official_apply_url

若 `unique_hash` 相同但 `content_hash` 变化，则说明职位内容更新。

## 索引建议

1. jobs(title)
2. jobs(city_name)
3. jobs(company_id)
4. jobs(status)
5. jobs(first_seen_at)
6. jobs(last_seen_at)
7. jobs(unique_hash unique)
8. user_saved_searches(user_id)
9. notifications(user_id, is_read)

## MVP 最小表集

如果先做最小版本，只保留以下表即可：

1. users
2. companies
3. jobs
4. user_favorite_companies
5. user_saved_searches
6. notifications

这样就已经足够完成搜索、收藏、提醒三大核心能力。
